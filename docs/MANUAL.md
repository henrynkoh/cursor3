# Facilitator manual — Seattle Cursor Vibe Week

Use this document to **run the cohort**: prep, daily flow, roles, and wrap-up. The live curriculum content is on the website (Days 1–7); this manual is **how** to execute it.

## Audience and outcome

- **Who:** 8–12 early-stage Seattle-area startups; 1–2 technical leads per team; builders using React/Next.js, Python, etc.
- **Outcome:** Teams ship real PRs with Cursor 3 (browser tool, subagents, Git), compare tools vs. budget on Day 5, and demo on Day 7.

## Roles

| Role | Responsibility |
|------|------------------|
| **Lead facilitator** | Schedule, demos, Day 5 discussion, Day 7 MC |
| **TA(s) 1–2** | Setup help, Cursor/Git issues, timekeeping |
| **Host venue** | Wi‑Fi, power, screen, room for gallery walk (Day 2) and demos (Day 7) |

## Pre-program (2 weeks out)

1. Confirm **Cursor 3** access (Pro/Team); share pricing transparency.
2. Open **Slack or Discord** channels: `#general`, `#wins`, `#blocked`, optional `#prompts`.
3. Send [QUICKSTART.md](QUICKSTART.md) to any internal dev running the curriculum site; participants only need **Cursor + GitHub + their repo**.
4. Book **hybrid venue** (Seattle) + AV; send calendar holds for 4–6 h/day core time.
5. Optional: shared **Notion** with links to `/playbook` prompts and a simple **token/time log** spreadsheet for Days 1–5.

## Day 0 (evening before)

- Post **agenda link** (your deployed site or PDF export).
- Reminder: laptop charger, Cursor updated, repo cloned.

## Daily rhythm (template)

| Block | Suggested time | Activities |
|--------|----------------|------------|
| **Morning** | ~2–3 h | Review prior homework, demo, paired work |
| **Afternoon** | ~2–3 h | Gallery, PR review, or sprint checkpoints |
| **Async** | Evening | Homework from site; log metrics |

Adjust for your cohort; keep **one shipped artifact** per team moving forward by Day 4.

## Day-by-day facilitator notes

### Day 1 — Fundamentals

- Keep **watch party** to 20–30 min; discussion: UI vs. terminal-first tools.
- End with **homework**: one refactor + log tokens/time.

### Day 2 — Browser tool

- Ensure everyone can run **embedded browser** on a real page (landing or dashboard).
- **Gallery walk:** strict timebox (e.g. 5 min/team).

### Day 3 — Subagents

- Set **scope limits** (paths, no prod secrets). Document findings in shared doc.

### Day 4 — GitHub

- Require a **named branch** and real PR; peer review across startups.

### Day 5 — Cost / policy

- Collect **usage stories** beforehand. Outcome: **90-day AI stack policy** draft per team.

### Day 6 — Sprint

- **2 h checkpoints**; cut scope if needed. Document prompts for Day 7 demo.

### Day 7 — Showcase

- **10–15 min/startup**: demo, workflow, metrics, surprises.
- Closing: certificates, shared folder, **1-month check-in** date.

## Success metrics (from curriculum)

Track qualitatively during the week:

- At least **one PR per participant** via full Cursor workflow.
- **2–4×** self-reported lift on UI-heavy work (with examples).
- Clear **adopt / not adopt** decision for Cursor 3 next quarter.
- **2–3** cross-startup follow-ups (intros, coffee, partnerships).

## Risk and mitigation

| Risk | Mitigation |
|------|------------|
| Tooling variance | TAs focus on Cursor install + repo access first morning |
| Scope creep on Day 6 | Pre-approve one scoped initiative per team on Day 5 evening |
| Quiet cohort | Use breakout pairs and cross-startup pairs early (Day 2) |

## Post-program
